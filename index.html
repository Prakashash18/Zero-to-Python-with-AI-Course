How to ask: a good code prompt

A good prompt is the whole skill. Three habits get you most of the way:

  1. Be specific and complete. Say the goal, the inputs, and the output you want.
  2. Give context and purpose. Tell it you're a beginner and why, so it explains at the right level.
  3. Ask one thing at a time. Get it working, then ask for a single improvement.

True or false?

Idea 02 · Conditional · 1 of 4

      
Which path runs?

      
The if checks a True/False question and picks one path.

      
steps = 12
if steps >= 10:
    print("active")
else:
    print("keep going")

      

        
        
It prints active. Is 12 >= 10? Yes — True — so the if path runs.

      

    


    
    

      
Idea 02 · Conditional · 2 of 4

      
And with fewer steps?

      
Same code, smaller number.

      
steps = 7
if steps >= 10:
    print("active")
else:
    print("keep going")

      

        
        
It prints keep going. 7 >= 10 is False, so the else path runs instead.

      

    


    
    

      
Idea 02 · Conditional · 3 of 4

      
Right on the line

      
What about exactly 10? This is where >= matters.

      
steps = 10
if steps >= 10:
    print("active")
else:
    print("keep going")

      

        
        
It prints active. >= means "greater than or equal", so 10 counts. (Plain > would have printed "keep going".)

      

    


    
    

      
Idea 02 · Conditional · 4 of 4

      
An if with no else

      
You don't always need an else. What prints here?

      
steps = 3
if steps >= 10:
    print("active")
print("done")

      

        
        
It prints only done. 3 >= 10 is False, so the indented line is skipped — but the last line isn't indented, so it always runs.

How many lines print?

      
This time it prints the same word each pass.

Idea 03 · for · 3 of 4

      
Adding up in a loop

      
A loop is how you build a running total — exactly like adding up daily steps.

      
total = 0
for n in range(4):
    total = total + n
print(total)

      

        
        
It prints 6. It adds 0 + 1 + 2 + 3 as the loop goes, then prints the total once at the end.

A for loop can walk through a list of things, not just numbers.

It prints Mon Tue Wed. Each time round, day becomes the next item — this is how you'll walk your step data in Step 3.

Idea 03 · while · 2 of 4

      
What's the final value?

      
Here it counts silently, then prints once at the end.

      
steps = 0
while steps < 5:
    steps = steps + 1
print(steps)

      

        
        
It prints 5. It keeps adding 1 while steps < 5, stopping the moment steps reaches 5.

      

    


    
    

      
Idea 03 · while · 3 of 4

      
What if it's false at the start?

      
A while checks its condition before running the body.

      
n = 10
while n < 5:
    print(n)
print("done")

      

        
        
It prints only done. 10 < 5 is already False, so the loop body never runs even once.

      

    


    
    

      
Idea 03 · while · 4 of 4

      
When does this stop?

      
Read carefully before you reveal — this one's a trap.

      
n = 1
while n > 0:
    n = n + 1

      

        
        
Never. n only grows, so n > 0 stays True forever — an infinite loop. Always make sure the condition can eventually become False. (That's why there's no "run" button here!)

      

      
The micro:bit loop. Your step counter uses while True: on purpose — it should never stop watching for shakes.

Same machine, new input

      
Build it once, reuse it — that's the whole point of a function.

      
def double(n):
    return n * 2

print(double(50))

      

        
        
It prints 100. Same machine, different input — no need to rewrite anything.

      

    


    
    

      
Idea 04 · Function · 3 of 4

      
Steps into distance

      
Here's the one you'll use later — about 0.75 m per step.

      
def steps_to_distance(steps):
    return steps * 0.75

print(steps_to_distance(2000))

      

        
        
It prints 1500.0. 2000 × 0.75 = 1500 metres. (Python shows .0 because the answer is a decimal number.)

      

    


    
    

      
Idea 04 · Function · 4 of 4

      
Reuse it twice

      
Call the same function as many times as you like.

      
def steps_to_distance(steps):
    return steps * 0.75

print(steps_to_distance(2000))
print(steps_to_distance(8000))

      

        
        
It prints 1500.0 then 6000.0. One machine, two inputs, two results — written once, used forever.

Step 2 · How it senses

      
How an accelerometer works

      
Inside the micro:bit is an accelerometer — the same motion sensor that lets your phone know when you tilt it. It measures movement along three directions:

      
    
        
  • X — tilting left and right
    • 
        
  • Y — tilting forward and back
    • 
        
  • Z — moving up and down
    • 
      

      
The micro:bit reads these values many times a second. When you take a step, your body bounces slightly — and that bounce shows up as a little spike in the readings. Count the spikes, and you've counted steps.

Step 2 · Code Fast

      
2 · Build the step counter

      
Every line is a Step 1 idea in the wild — a variable, a forever while loop, and an if.

      
from microbit import *

steps = 0              # the variable (a box)
while True:            # the loop (runs forever)
    if accelerometer.was_gesture('shake'):  # the if (a step!)
        steps += 1     # add one
        display.show(steps)

      
How shake works. accelerometer.was_gesture('shake') watches those X/Y/Z readings and returns True when it sees a sharp bounce. Each detected shake counts as roughly one step — simple, and close enough to start.

Step 3 · Build with Data

      
2 · Total and average

      
sum() adds every number in the list; len() counts them. Divide to get the average, and round() tidies the decimal.

      
steps = [6200, 8100, 6400, 9100, 7300, 15630, 13200]

print("Total:", sum(steps))
print("Average:", round(sum(steps) / len(steps)))

      

        

        

          
          

          

        

      

    


    
    

      
Step 3 · Build with Data

      
3 · Best, worst — and which day

      
max() and min() find the biggest and smallest. But a number alone isn't the story — .index() finds where it sits, so we can read the matching day.

      
days  = ["Mon","Tue","Wed","Thu","Fri","Sat","Sun"]
steps = [6200, 8100, 6400, 9100, 7300, 15630, 13200]

print("Best:", max(steps), " Worst:", min(steps))
best = steps.index(max(steps))
print("Best day:", days[best])

      

        

        

          
          

          

        

      

    


    
    

      
Step 3 · Build with Data

      
4 · Draw a quick chart

      
A for loop can turn each number into a row of # marks — a bar chart with no chart library at all. (// 1000 means "one # per 1000 steps".)

      
days  = ["Mon","Tue","Wed","Thu","Fri","Sat","Sun"]
steps = [6200, 8100, 6400, 9100, 7300, 15630, 13200]

for i in range(len(days)):
    bar = "#" * (steps[i] // 1000)
    print(days[i], bar, steps[i])
